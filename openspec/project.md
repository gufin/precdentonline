# Project Context

## Purpose
Веб-приложение "Прецедент Онлайн" для поиска и анализа юридических прецедентов (судебных решений) из базы данных ГАС "Правосудие". Предоставляет как семантический (по смыслу), так и классический (по ключевым словам) поиск с расширенными возможностями фильтрации.

**Основные цели:**
- Быстрый доступ к базе судебных решений
- Интеллектуальный семантический поиск для точных результатов
- Удобная фильтрация по множеству параметров (суд, судья, даты, стороны, уровень суда)
- Современный и понятный интерфейс для юристов

## Tech Stack
- **Frontend:** React 19.2, TypeScript 5.8
- **Build Tool:** Vite 6.2
- **Styling:** Tailwind CSS 3.4 + PostCSS
- **Backend:** Node.js + Express 4.22
- **Runtime:** Node.js 18+
- **Package Manager:** npm

## Project Conventions

### Code Style
- TypeScript для типобезопасности
- Функциональные компоненты React с хуками (useState, useEffect, useMemo)
- Именование: PascalCase для компонентов, camelCase для функций и переменных
- Интерфейсы TypeScript в `types.ts` для централизованного управления типами
- Tailwind классы для стилизации (избегать inline styles)
- Комментарии на русском языке в коде

### Architecture Patterns
- **Разделение на слои:**
  - `components/` - переиспользуемые UI компоненты
  - `services/` - бизнес-логика и API взаимодействие
  - `types.ts` - определения типов
  - `App.tsx` - главный компонент приложения
  
- **State Management:** React hooks (useState) для локального состояния
- **Data Flow:** Props для передачи данных, callbacks для событий
- **API Communication:** Централизованный сервисный слой (`services/api.ts`)
- **Error Handling:** Try-catch с понятными сообщениями пользователю

### Testing Strategy
**Текущее состояние:** Тесты не реализованы

**Планируемый подход:**
- Unit тесты для критических функций (фильтрация, сортировка)
- Component тесты для UI компонентов
- E2E тесты для основных пользовательских сценариев (поиск, фильтрация)
- Приоритет: покрытие API клиента и трансформации данных

### Git Workflow
**Важно:** Проект использует SSH-передачу файлов для деплоя, а НЕ Git workflows. Измененные файлы передаются через `scp` на сервер.

**Локальная разработка:**
- Работа в основной ветке или feature-ветках
- Коммиты с понятными сообщениями на русском
- Перед коммитом: проверка линтера и сборки (`npm run build`)

## Domain Context

### Юридическая специфика
- **Судебные решения:** Основная сущность - документ суда с метаданными
- **Стороны дела:** Истцы (Plaintiffs), Ответчики (Defendants), Третьи лица, Прочие
- **Иерархия судов:** Первая инстанция → Апелляция → Кассация
- **Категории дел:** Гражданские, административные, уголовные и т.д.
- **Вступление в силу:** Важный параметр - дата вступления решения в законную силу

### Форматы данных
- **Даты:** dd.mm.yyyy (русский формат)
- **Номера дел:** Уникальный идентификатор дела в формате суда
- **GUID:** Уникальный идентификатор документа в ГАС Правосудие

## Important Constraints

### Технические
- Семантический поиск требует внешний сервис (может быть недоступен)
- API ключи должны быть защищены (использование .env и proxy через backend)
- Размер ответа от классического API: до 30 документов за запрос
- Пагинация реализована на клиенте (12 карточек на страницу)

### Бизнес-ограничения
- База данных содержит решения судов общей юрисдикции (ГАС Правосудие)
- **Планируемая монетизация:** Лимиты на количество поисков + платная подписка
- **Будущие фичи:** AI-ассистент для анализа решений, авторизация, биллинг

### Безопасность
- API ключи НЕ должны быть видны в клиентском коде
- Все запросы к внешним API идут через Express прокси
- Environment variables в `.env` файле (не коммитить!)

## External Dependencies

### APIs
1. **Классический поиск API**
   - Base URL: `https://api.you-right.ru/gas`
   - Требуется: API_KEY
   - Endpoints: `/search`, `/get_Count_Records`

2. **Семантический поиск API**
   - Base URL: `http://195.35.56.180`
   - Требуется: SEMANTIC_API_KEY
   - Endpoint: `/search`
   - Особенности: Поиск по смыслу запроса, возвращает релевантность (score)

3. **Gemini API** (опционально)
   - Для будущего AI-ассистента
   - Требуется: GEMINI_API_KEY

### Сервисы
- **ГАС "Правосудие":** Государственная автоматизированная система судов
- Ссылки на дела ведут на сайты судов (kad.arbitr.ru и др.)
