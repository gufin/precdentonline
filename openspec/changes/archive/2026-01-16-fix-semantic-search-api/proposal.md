# Change: Исправление семантического поиска и корректировка фильтров

## Why
Семантический поиск не работает из-за неправильного URL API и несоответствия фильтров между клиентом и API. Текущий код использует `http://195.35.56.180`, а должен использовать `https://semsearch.ru/`. Также фильтры не соответствуют спецификации API - отсутствует поддержка `categories`, а некоторые фильтры из классического поиска ошибочно применяются к семантическому.

## What Changes
- Изменение базового URL семантического API с `http://195.35.56.180` на `https://semsearch.ru/`
- Добавление нового фильтра `categories` (массив категорий дел) в интерфейс `SearchFilters`
- Обновление типов для поддержки поля `categories` в результатах поиска
- Документирование того, какие фильтры доступны для семантического поиска vs классического поиска

## Impact
- Affected specs: `semantic-search` (новая спецификация)
- Affected code:
  - `services/api.ts` - изменение константы `semanticApiBase`
  - `types.ts` - добавление поля `categories` в `SearchFilters` и `SearchResultItem`
  - `.env` файлы - обновление переменной окружения `SEMANTIC_API_BASE`
  - `server.js` - обновление переменной окружения `SEMANTIC_API_BASE`
