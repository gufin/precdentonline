# Change: Добавление автоматического деплоя через GitHub Actions

## Why
В данный момент деплой выполняется вручную через скрипт `deploy.sh` с локальной машины разработчика. Это требует установки `lftp`, хранения FTP credentials локально и ручного запуска при каждом изменении. Автоматизация деплоя при пуше в ветку main повысит скорость доставки изменений, снизит риск человеческих ошибок и обеспечит консистентность процесса деплоя.

## What Changes
- Добавление GitHub Actions workflow для автоматического деплоя при пуше в ветку `main`
- Настройка GitHub Secrets для безопасного хранения FTP credentials
- Автоматическая сборка проекта (`npm run build`) в CI/CD pipeline
- Автоматическая загрузка собранных файлов на FTP сервер
- Создание резервных копий перед деплоем
- Уведомления о статусе деплоя
- Документация процесса настройки и использования

## Impact
- **Affected specs:** Создается новая capability `ci-cd-pipeline`
- **Affected code:**
  - `.github/workflows/deploy.yml` (новый файл) - основной workflow для автодеплоя
  - `README.md` - добавление раздела о CI/CD и GitHub Actions
  - `openspec/project.md` - обновление раздела Git Workflow
- **Dependencies:** 
  - Требуется настройка GitHub Secrets в репозитории
  - Существующий `deploy.sh` останется для локального деплоя
  - Совместимость с текущим процессом деплоя через FTP
- **Breaking changes:** Нет
