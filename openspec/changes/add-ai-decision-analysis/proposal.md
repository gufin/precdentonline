# Change: Добавление AI-анализа судебных решений

## Why
Пользователи нуждаются в быстром понимании сути судебного решения без чтения полного текста. AI-анализ предоставляет структурированное резюме с ключевыми фактами: фабула дела, позиции сторон, мотивировка суда и резолютивная часть с суммами.

Кнопка "AI Анализ" уже присутствует в интерфейсе модального окна (Modal.tsx:174-177), но функционал не реализован.

## What Changes
- Интеграция с Recognition API (dev.you-right.ru) для получения структурированного анализа решений
- Асинхронная обработка запросов с polling механизмом (POST для запуска → GET для проверки статуса)
- UI для отображения статуса загрузки (spinner) во время обработки запроса
- Красивое форматирование результатов анализа в стиле сайта
- Обработка ошибок и таймаутов API
- Использование HTTPS вместо HTTP для всех API вызовов
- Backend proxy endpoint для безопасного хранения API ключей

## Impact
- **Affected specs:** Новая capability `ai-analysis`
- **Affected code:**
  - `components/Modal.tsx` - добавление обработчика кнопки AI Анализ и UI для результатов
  - `services/api.ts` - новые функции для работы с Recognition API
  - `server.js` - новый proxy endpoint для Recognition API
  - `types.ts` - новые типы для AI анализа
- **External dependencies:** Recognition API (https://dev.you-right.ru/ur_ai_api/hs/RecognitionService/recognize)
- **Breaking changes:** Нет

## User Flow
1. Пользователь открывает модальное окно с судебным решением
2. Нажимает кнопку "AI Анализ"
3. Видит индикатор загрузки (крутилку) с сообщением "Анализируем решение..."
4. Система отправляет текст решения в Recognition API
5. Polling проверяет статус каждые 2-3 секунды
6. После завершения обработки отображается структурированный результат:
   - Супер-краткая фабула дела
   - Позиции и доводы сторон (истец, ответчик)
   - Мотивировка суда
   - Резолютивная часть с суммами
7. При ошибке показывается понятное сообщение с возможностью повторить попытку
