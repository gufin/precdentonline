# Capability: AI-анализ судебных решений

## ADDED Requirements

### Requirement: AI Analysis Initiation
Система ДОЛЖНА (SHALL) предоставлять возможность запустить AI-анализ судебного решения из модального окна просмотра решения.

#### Scenario: Пользователь запускает анализ
- **WHEN** пользователь открывает модальное окно с судебным решением
- **AND** нажимает кнопку "AI Анализ"
- **THEN** система отправляет текст решения в Recognition API
- **AND** отображает индикатор загрузки с сообщением "Анализируем решение..."
- **AND** начинает polling для проверки статуса обработки

#### Scenario: Полный текст недоступен
- **WHEN** пользователь нажимает кнопку "AI Анализ"
- **AND** полный текст решения недоступен (loadingText=false, fullText=null, textError!=null)
- **THEN** система показывает сообщение "Полный текст решения недоступен для анализа"
- **AND** не отправляет запрос в API

### Requirement: Asynchronous Processing
Система ДОЛЖНА (SHALL) обрабатывать AI-анализ асинхронно с использованием polling механизма.

#### Scenario: Успешная обработка
- **WHEN** система получает requestID от Recognition API
- **THEN** система начинает polling каждые 2 секунды
- **AND** проверяет статус через GET /api/ai-analysis/status?requestID={id}
- **WHEN** статус становится "Завершен"
- **THEN** система отображает результаты анализа
- **AND** прекращает polling

#### Scenario: Обработка в процессе
- **WHEN** система получает статус отличный от "Завершен" и "Ошибка"
- **THEN** система продолжает polling
- **AND** увеличивает интервал до 3 секунд после 5 попыток
- **AND** увеличивает интервал до 5 секунд после 15 попыток

#### Scenario: Таймаут обработки
- **WHEN** прошло более 60 секунд (20 попыток polling)
- **AND** статус все еще не "Завершен"
- **THEN** система прекращает polling
- **AND** показывает сообщение "Анализ занял слишком много времени. Попробуйте позже"
- **AND** предоставляет кнопку "Повторить попытку"

### Requirement: Result Display
Система ДОЛЖНА (SHALL) отображать результаты AI-анализа в структурированном и читаемом виде.

#### Scenario: Отображение успешного результата
- **WHEN** система получает результат с status="Завершен"
- **AND** result.СтрокаJson содержит валидный JSON
- **THEN** система парсит JSON и отображает следующие секции:
  - "Супер-краткая фабула дела" - с выделением как основной блок
  - "Позиции и доводы сторон" - разделенные на истца и ответчика
  - "Мотивировка суда" - как текстовый блок
  - "Резолютивная часть" - с форматированием сумм и победителя
- **AND** применяет стилизацию в соответствии с темной темой сайта

#### Scenario: Форматирование сумм
- **WHEN** в резолютивной части присутствуют денежные суммы
- **THEN** система форматирует суммы с разделителями тысяч (пробелы)
- **AND** выделяет итоговую сумму жирным шрифтом
- **AND** добавляет символ "руб." после каждой суммы

#### Scenario: Responsive отображение
- **WHEN** пользователь просматривает результаты на мобильном устройстве
- **THEN** система адаптирует layout для узких экранов
- **AND** сохраняет читаемость всех секций

### Requirement: Error Handling
Система ДОЛЖНА (SHALL) обрабатывать все возможные ошибки при выполнении AI-анализа.

#### Scenario: Ошибка сети
- **WHEN** запрос к Recognition API не может быть выполнен (network error)
- **THEN** система показывает сообщение "Не удалось подключиться к сервису анализа"
- **AND** предоставляет кнопку "Повторить попытку"

#### Scenario: Ошибка от API
- **WHEN** Recognition API возвращает status="Ошибка"
- **OR** result.ОписаниеОшибки не пустое
- **THEN** система показывает сообщение из ОписаниеОшибки
- **AND** предоставляет кнопку "Повторить попытку"

#### Scenario: Ошибка парсинга результата
- **WHEN** result.СтрокаJson не является валидным JSON
- **OR** отсутствуют обязательные поля в результате
- **THEN** система показывает сообщение "Не удалось обработать результат анализа"
- **AND** логирует ошибку в console для отладки

#### Scenario: Повторная попытка после ошибки
- **WHEN** пользователь нажимает кнопку "Повторить попытку"
- **THEN** система сбрасывает состояние ошибки
- **AND** запускает новый запрос анализа
- **AND** отображает индикатор загрузки

### Requirement: Backend Proxy
Система ДОЛЖНА (SHALL) использовать backend proxy для безопасного взаимодействия с Recognition API.

#### Scenario: Запуск анализа через proxy
- **WHEN** frontend отправляет запрос на запуск анализа
- **THEN** запрос идет на POST /api/ai-analysis/recognize
- **AND** backend добавляет Authorization header с Bearer token
- **AND** отправляет запрос к https://dev.you-right.ru/ur_ai_api/hs/RecognitionService/recognize
- **AND** возвращает requestID frontend'у

#### Scenario: Проверка статуса через proxy
- **WHEN** frontend отправляет запрос проверки статуса
- **THEN** запрос идет на GET /api/ai-analysis/status?requestID={id}
- **AND** backend добавляет Authorization header с Bearer token
- **AND** отправляет запрос к https://dev.you-right.ru/ur_ai_api/hs/RecognitionService/recognize?requestID={id}
- **AND** возвращает результат frontend'у

#### Scenario: Безопасность API ключей
- **WHEN** любой запрос к Recognition API выполняется
- **THEN** Authorization token НЕ ДОЛЖЕН быть виден в frontend коде
- **AND** token ДОЛЖЕН храниться в переменных окружения backend (.env)
- **AND** token ДОЛЖЕН добавляться только на backend в server.js

### Requirement: Loading State UI
Система ДОЛЖНА (SHALL) предоставлять визуальную индикацию процесса анализа.

#### Scenario: Индикатор загрузки
- **WHEN** анализ находится в процессе выполнения
- **THEN** система отображает анимированный spinner
- **AND** показывает сообщение "Анализируем решение..."
- **AND** блокирует повторное нажатие кнопки "AI Анализ"

#### Scenario: Прогресс polling
- **WHEN** система выполняет polling запросы
- **THEN** индикатор загрузки остается видимым
- **AND** пользователь может закрыть модальное окно (polling прекращается)

### Requirement: API Request Format
Система ДОЛЖНА (SHALL) формировать запросы к Recognition API в правильном формате.

#### Scenario: Формат запроса на запуск анализа
- **WHEN** система отправляет запрос на запуск анализа
- **THEN** тело запроса ДОЛЖНО содержать:
  - recognitionFunctionName: "Резюмировать решение"
  - fileData: полный текст судебного решения
- **AND** Content-Type ДОЛЖЕН быть "application/json"
- **AND** URL ДОЛЖЕН использовать HTTPS протокол

#### Scenario: Формат запроса проверки статуса
- **WHEN** система отправляет запрос проверки статуса
- **THEN** requestID ДОЛЖЕН передаваться как query параметр
- **AND** метод ДОЛЖЕН быть GET
- **AND** URL ДОЛЖЕН использовать HTTPS протокол
