# Change: Добавление Docker-контейнеризации для локальной разработки

## Why
В настоящее время локальная разработка требует ручной установки Node.js 18+, npm пакетов и настройки окружения. Это создаёт проблемы при работе в команде (разные версии Node.js, различия в окружении) и затрудняет онбординг новых разработчиков. Контейнеризация обеспечит консистентное окружение разработки для всех участников проекта.

## What Changes
- Добавление Dockerfile для сборки образа приложения
- Добавление docker-compose.yml для оркестрации frontend и backend сервисов
- Настройка volume мounts для hot-reload в режиме разработки
- Интеграция с существующей системой переменных окружения (.env)
- Документация по запуску приложения через Docker
- Оптимизация для development-режима (многоступенчатая сборка не требуется, фокус на скорости)

## Impact
- Affected specs: `local-dev-environment` (новая capability)
- Affected code: 
  - Корневая директория проекта (новые файлы: Dockerfile, docker-compose.yml, .dockerignore)
  - README.md (добавление инструкций по Docker)
- Не затрагивает существующий процесс деплоя через SSH
- Не меняет логику приложения
- Полностью обратно совместимо с текущим способом разработки (npm run dev:all)
